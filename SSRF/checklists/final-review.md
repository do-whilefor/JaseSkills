# 最终反向审查清单

> 用于报告交付前反向审判。目标是降级证据不足结论，清理误报，补齐盲区。

## 1. confirmed 证据反查

对每一个 confirmed 逐项检查：

- [ ] 是否真的有 canary 回连。
- [ ] canary 请求是否来自目标应用服务端，而不是浏览器。
- [ ] canary 请求是否不是测试工具自己发出的。
- [ ] canary 请求是否不是 redirector 自身请求。
- [ ] marker 是否唯一。
- [ ] 请求时间是否与触发时间对应。
- [ ] 应用日志是否对应同一个 case-id 或同一触发窗口。
- [ ] 是否有完整请求样本。
- [ ] 是否有响应样本。
- [ ] 是否有代码调用链。
- [ ] 是否有正例。
- [ ] 是否有反例。
- [ ] 是否有 blocked case。
- [ ] 是否能用最小步骤复现。
- [ ] 是否能写成回归测试。

处理规则：任一项为否，降级为 candidate 或 needs_review，并写明缺哪条证据。

## 2. 暴露面补漏审查

- [ ] 管理后台连接测试。
- [ ] Webhook 测试、重试、失败重放。
- [ ] 用户头像远程导入。
- [ ] 图片代理和缩略图生成。
- [ ] OpenGraph / link preview。
- [ ] RSS / Atom 订阅。
- [ ] Markdown / HTML / SVG 预览。
- [ ] PDF 生成。
- [ ] 邮件模板预览。
- [ ] 页面截图功能。
- [ ] Headless browser 渲染。
- [ ] Git 仓库导入。
- [ ] 第三方集成配置。
- [ ] OAuth/OIDC discovery。
- [ ] SAML metadata 导入。
- [ ] 插件、主题、模板远程安装。
- [ ] 数据源连接器。
- [ ] 对象存储自定义 endpoint。
- [ ] 代理配置。
- [ ] 任务队列中延迟触发的 URL。
- [ ] 定时同步任务。
- [ ] 失败重试队列。
- [ ] 审批流、通知流、报表流中的 URL。
- [ ] GraphQL mutation 中的 URL。
- [ ] WebSocket 消息中的 URL。
- [ ] multipart 字段里的 URL。
- [ ] YAML/JSON/XML 导入里的深层 URL。
- [ ] 前端没有字段但后端 DTO/model 接受的 URL。
- [ ] 移动端 API 或旧版 API。
- [ ] 测试接口、debug 接口、internal API 但仍在本地项目中暴露的入口。

## 3. 偏门链路审查

- [ ] 是否存在保存时不请求、预览时才请求的保存型 SSRF。
- [ ] 是否存在普通用户提交、管理员页面打开后触发的二次 SSRF。
- [ ] 是否存在租户 A 配置、租户 B worker 执行的跨租户 SSRF。
- [ ] 是否存在低权限用户配置 webhook、高权限系统任务执行的权限放大 SSRF。
- [ ] 是否存在导入文件中远程资源被解析器自动请求。
- [ ] 是否存在 HTML/PDF/图片处理库自动加载外链。
- [ ] 是否存在跳转前校验、跳转后不校验。
- [ ] 是否存在字符串校验通过、URL parser 解析后变形。
- [ ] 是否存在 DNS 校验和连接时解析不一致。
- [ ] 是否存在只校验 host、不校验最终 IP。
- [ ] 是否存在只校验创建时 URL、不校验执行时 URL。
- [ ] 是否存在前端限制 URL、后端没有限制。
- [ ] 是否存在后台任务绕过同步接口校验。
- [ ] 是否存在测试连接接口比正式保存接口更弱。
- [ ] 是否存在重试、回放、导入、同步、预览、导出中的非主流程入口。

## 4. 代码与依赖反查

- [ ] 已列出所有 HTTP 客户端依赖。
- [ ] 已列出所有 URL parser 依赖。
- [ ] 已列出所有会自动加载远程资源的库。
- [ ] 已列出所有支持 redirect 的库。
- [ ] 已列出所有异步任务 worker。
- [ ] 已列出所有文件解析器。
- [ ] 已列出所有 HTML/PDF/image 渲染组件。
- [ ] 已列出所有 cloud/storage/webhook/OAuth/SAML 集成组件。
- [ ] 对每个组件判断用户是否能影响目标地址。
- [ ] 对每个组件判断是否有最终地址校验。
- [ ] 对每个组件判断是否有 redirect 后校验。
- [ ] 对每个组件判断是否有 DNS/IP 校验。
- [ ] 对每个组件判断是否有协议白名单。
- [ ] 对每个组件判断是否有租户/权限约束。
- [ ] 对每个组件判断是否有日志审计。

## 5. 强制补测优先级

P0 必须优先处理：

- [ ] 低权限可触发服务端请求。
- [ ] 异步 worker SSRF。
- [ ] 管理员预览二次 SSRF。
- [ ] webhook 测试 SSRF。
- [ ] PDF/HTML/图片渲染 SSRF。
- [ ] redirect 后校验缺失。
- [ ] 多租户/多角色组合 SSRF。

P1 必须形成补测计划：

- [ ] 导入文件触发。
- [ ] RSS/OpenGraph/link preview。
- [ ] GraphQL/WebSocket/multipart 深层 URL。
- [ ] OAuth/OIDC/SAML 配置。
- [ ] 对象存储自定义 endpoint。
- [ ] 旧版 API 和隐藏 API。

P2 必须列入下一轮：

- [ ] 缓存、重试、错误处理。
- [ ] 日志泄露。
- [ ] 限速缺失。
- [ ] 可观测性缺失。
- [ ] 修复回归测试缺失。

## 6. 最终追责问题

每个问题必须回答“是/否、证据路径、缺口、补测步骤、预期结果、修复后的结论等级”：

- [ ] 是否只看显眼 URL 参数而漏掉隐式远程资源加载。
- [ ] 是否只审同步接口而漏掉 worker、队列、定时任务、失败重试。
- [ ] 是否只测普通用户路径而漏掉管理员预览、后台审核、系统任务触发。
- [ ] 是否只看前端字段而漏掉后端 DTO/model 隐藏字段。
- [ ] 是否只测 REST 而漏掉 GraphQL、WebSocket、multipart、导入文件。
- [ ] 是否只测直接请求而漏掉 redirect、DNS、URL 规范化、parser 差异。
- [ ] 是否只看业务代码而漏掉依赖库自动加载远程资源。
- [ ] 是否只证明能提交 URL，却没有证明服务端真的请求 URL。
- [ ] 是否把浏览器请求误判成服务端请求。
- [ ] 是否缺少反例和 blocked case。
- [ ] 是否缺少证据文件。
- [ ] 是否缺少回归测试。

## 7. 最终输出清理

- [ ] 原报告中被降级的结论已列出。
- [ ] 原报告中证据不足的结论已列出。
- [ ] 新发现 candidate 已列出。
- [ ] 新确认 confirmed 已列出。
- [ ] 新发现 blocked 已列出。
- [ ] 仍未验证 needs_review 已列出。
- [ ] 最危险的 5 条 SSRF 链路只包含有证据或明确 needs_review 的链路。
- [ ] 最容易被忽略的 10 个入口已列出。
- [ ] 最可能产生误报的 10 个点已列出。
- [ ] 下一轮必须动态验证清单已列出。
- [ ] 修复优先级已列出。
- [ ] 回归测试清单已列出。
- [ ] 证据索引已列出。
