# 原文能力与基于文档延伸的隔离规则

## 原文能力
来自用户文档的能力包括：授权边界、输入确认、四线交叉验证、资产账本、运行态入口、代码驱动路由、动态验证、信息类型分类、小众偏门信息面、三轮反思、证据标准、最终输出格式、质量门槛、二轮反思模式和十种创新方法。

##
以下内容是工程化增强，不得伪装为原文直接要求：QG-01 到 QG-10 编号体系、Playwright/MCP 浏览器存储采集脚本、Docker 只读考古脚本、GraphQL 非破坏 probe 脚本、资产影子账本 diff 脚本、报告自动评分器、部署面只读 inventory 脚本、包产物只读 inventory 脚本、协议差异模板、证据链压缩摘要、三档执行路径。

这些增强只能服务于原文目标，不能降低、替换或覆盖原文约束。

## 延伸边界

- `08-skill-adversarial-audit-regression` 是维护审查能力，不是原始信息暴露面审计目标本身。
- `templates/unified-quality-gate.md` 是对原始质量门槛的工程化门禁，不新增危险验证方式。
- `examples/regression-test-suite.md` 是测试体系延伸，用于防止回归。
- 所有 延伸不得被写成原始文档的直接要求；报告中必须标记“基于文档延伸”。
