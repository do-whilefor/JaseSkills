# 信息暴露面反思 Checklist

## 第一轮：遗漏反思

| 问题 | 结果 | 是否遗漏 | 下一步最小验证 |
|---|---|---|---|
| 我刚才主要依赖了哪些入口？ |  |  |  |
| 哪些入口完全没覆盖？ |  |  |  |
| 是否只看源码，没有看运行态？ |  |  |  |
| 是否只看 API，没有看浏览器？ |  |  |  |
| 是否只看未认证，没有做角色差异？ |  |  |  |
| 是否忽略构建产物、缓存、日志、导出文件？ |  |  |  |
| 是否忽略容器、镜像、volume、临时目录？ |  |  |  |
| 是否遗漏非 HTTP 服务？ |  |  |  |

## 第二轮：误报反思

| 发现编号 | 是否运行态可访问 | 是否包含敏感或非预期信息 | 是否测试数据 | 是否预期可见 | 是否有影响 | 是否稳定复现 | 当前状态 | 不可报告原因 |
|---|---|---|---|---|---|---|---|---|

## 第三轮：剑走偏锋反思

| 视角 | 检查内容 | 新增候选 | 是否需要动态验证 |
|---|---|---|---|
| 只看前端 JS | 接口、角色、feature flag、schema、source map |  |  |
| 只看容器 | image layer、volume、启动脚本、.env、日志 |  |  |
| 只看错误页面 | 404/405/500、GraphQL error、ORM 错误 |  |  |
| 只看缓存 | 浏览器、service worker、静态目录、导出目录 |  |  |
| 只看构建系统 | lockfile、CI、artifact、build path |  |  |
| 低权限用户 | 导出、搜索、自动补全、附件、日志 |  |  |
| 生命周期边界 | 邀请、登出、权限变更、成员移除 |  |  |
| 负空间 | 应鉴权但可能未鉴权的下载、预览、schema、metrics |  |  |
