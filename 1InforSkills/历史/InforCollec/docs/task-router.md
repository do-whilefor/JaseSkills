# 任务路由规则

## 先判断任务类型

1. 用户是否要求对本机授权项目执行信息暴露面验证？是：调用 00。否：继续判断。
2. 用户是否只要求概念解释？是：不调用执行型 Skill。
3. 用户是否要求公网、未知目标、破坏性测试、外部 token 验证、完整敏感信息？是：拒绝危险部分，要求收回本机授权范围。
4. 用户是否已有第一轮结果并要求反思？是：调用 06，再回流 04，最后 07。
5. 用户是否只给静态目录或源码？是：调用 03，所有结果为静态候选。
6. 用户是否只给 Base URL 或端口？是：调用 01→02→04。
7. 用户是否要求报告？是：调用 07；若缺动态证据，输出不可交付原因。

## 误触发样例

| 用户输入 | 是否调用 | 原因 |
|---|---|---|
| “什么是信息暴露？” | 否 | 概念解释，不是项目验证 |
| “帮我扫 example.com 信息泄露” | 否 | 公网未知目标，范围不明 |
| “把这个 token 完整打印出来” | 否 | 违反脱敏规则 |
| “当前 localhost:3000 项目做信息暴露动态验证” | 是 | 本机授权运行态任务 |
| “刚才结果还有什么遗漏？” | 是，调用 06 | 第一轮后反思补漏 |

本文件用于减少触发不稳定，不新增原文档之外的审计目标。

## 路由补充

| 用户信号 | 先调用 | 再调用 | 禁止点 |
|---|---|---|---|
| gRPC、RPC、proto、IDL、reflection、grpcurl | 04 | 06、07 | 默认不得调用业务方法；不得调用写入/删除/修改方法 |
| Playwright MCP、浏览器存储、IndexedDB、Cache Storage | 05 | 04、07 | 不输出完整 token/cookie；MCP 不可用时不得伪造执行 |
| 资产账本、JSON schema、机器校验交接 | 03 | 04、05、06、07 | 不符合 schema 的资产不得进入最终账本 |
| evidence manifest、证据清单 | 07 | 无 | manifest 不是漏洞证明，只是证据索引 |
| Skill 自测、维护检查 | 00/07 | 维护者说明 | 自测失败不得通过降低规则绕过 |
