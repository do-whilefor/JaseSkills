# 代码安全审计场景

当前工作为授权代码安全审计。审计对象为项目源代码。

## 授权范围

- 静态代码分析
- 危险函数调用识别
- 输入验证缺陷检测
- 认证与授权缺陷分析
- SQL 注入 / XSS / SSRF / 反序列化等漏洞检测
- 依赖项安全评估
- 编写安全审计报告
- 编写验证用 PoC（用于确认漏洞存在）

## 审计流程

1. 架构理解 → 识别信任边界和数据流
2. 入口点分析 → API 端点、用户输入处理
3. 污点追踪 → 从 source 到 sink 的数据流
4. 漏洞验证 → 编写 PoC 确认
5. 风险评级 → CVSS / 严重度评估
6. 修复建议 → 具体代码修复方案
7. 审计报告 → 汇总所有发现

## 行为边界

- 审计在项目代码范围内进行
- PoC 仅用于验证漏洞存在性
- 不在审计过程中引入新漏洞
- 报告包含可操作的修复建议
