# SecKB 质量门禁策略（v2）

## 状态规则

- `promoted`：通过所有硬性门槛，可作为高可信知识条目。
- `needs_review`：信息有价值但缺字段、缺交叉验证、缺版本、缺边界或缺证据。
- `conflict`：多个来源字段冲突，必须保留冲突字段和来源。
- `stale`：时间漂移、最后检查过旧、规则可能变更。
- `rejected`：不可信、不可验证、误报、越界、违反规则或来源不足。

## promoted 硬性门槛

一条记录要进入 `promoted`，必须同时满足：

1. 至少一个高可信来源。
2. 明确发布时间或更新时间。
3. 明确产品、版本或适用范围。
4. 明确影响。
5. 明确安全边界。
6. 明确误报条件。
7. 明确不可报告原因。
8. 明确修复建议。
9. 明确动态验证方式，且仅限本机或授权环境。
10. 有 tags 和索引入口。
11. 没有未解决来源冲突。
12. 不是单一 PoC、标题、截图、二次转载或 AI 摘要。

## 可报告性门槛

漏洞候选要进入可报告状态，必须满足：

1. 代码证据存在。
2. 动态证据存在。
3. 影响证据存在。
4. 修复建议存在。
5. 三次稳定复现。
6. 不在 SRC 禁止边界。
7. 影响证明不读取真实敏感数据。
8. evidence manifest 完整。

## 停止条件

任一条件命中即停止输出最终结论：

- 未授权目标。
- 需要 DoS、压测、批量探测、数据破坏、真实敏感数据读取、权限维持、横向移动。
- SRC 禁止边界。
- 只有工具告警。
- 只有错误页面。
- 只有 README 或测试数据关键词。
- 只有 PoC 仓库且无官方确认。
- 产品或版本不匹配。
