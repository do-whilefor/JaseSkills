# final-review

本清单用于报告输出前的最后复核。它不是可选步骤；未执行时，最终报告不得交付。

## 1. 第二轮反思

- [ ] 我是否只盯着 HTTP 接口，漏掉了 CLI、worker、cron、webhook、import/export？
- [ ] 我是否只 grep 了明显 API，漏掉 wrapper、adapter、helper、插件、事件监听器？
- [ ] 我是否漏掉了数据库字段二次触发命令执行？
- [ ] 我是否漏掉了文件名、压缩包条目名、metadata、content-type、输出路径？
- [ ] 我是否漏掉了参数注入，只关注 shell 注入？
- [ ] 我是否漏掉了多租户 worker 的权限错位？
- [ ] 我是否误把普通回显当成命令执行？
- [ ] 我是否有任何 confirmed 没有动态证据？
- [ ] 我是否对每个 confirmed 都做了负向验证？
- [ ] 我是否有高危结论但没有最小复现？
- [ ] 我是否只测试了 Linux，没考虑 Windows/macOS 参数解析差异？
- [ ] 我是否检查了依赖工具本身的危险参数？
- [ ] 我是否检查了项目中的隐藏参数、未文档化参数、后端接受但前端不暴露的参数？
- [ ] 我是否检查了失败重试、异步补偿、日志处理、告警通知这类边缘链路？
- [ ] 我是否检查了 admin-only 功能是否可被低权限用户间接触发？

处理规则：任一项回答为“未检查”时，回到对应阶段补查；无法补查时写入 `needs_environment` 或未完成验证表。

## 2. 第三轮反常规补挖

- [ ] 从所有外部工具调用反向找上游输入。
- [ ] 从所有用户可写数据库字段反向找后台消费点。
- [ ] 从所有文件落盘点反向找后续处理链。
- [ ] 从所有队列 topic/job name 反向找生产者。
- [ ] 从所有配置读取点反向找可写配置来源。
- [ ] 从所有插件/hook/extension 机制反向找用户可控入口。
- [ ] 从所有导入/恢复/迁移功能反向找命令链。
- [ ] 从所有 export/report/preview/render 功能反向找转换工具。
- [ ] 从所有 CI/build/test/devtool 调用反向找是否可由 Web 或项目配置触发。
- [ ] 从所有权限边界反向找低权限用户是否能污染高权限执行上下文。

处理规则：新增路径必须回到 source → transform → sink 数据流图和 replay plan；不得只写在反思里。

## 3. confirmed 逐项审判

对每一个 `confirmed` 逐项回答：

- [ ] 是否真的有命令执行证据，而不是普通字符串回显？
- [ ] marker 是否来自真实子进程执行结果？
- [ ] 是否存在 stdout、stderr、退出码、marker 文件、调用栈、请求响应、worker 日志中的至少两类交叉证据？
- [ ] 是否做了负向验证？
- [ ] 是否证明低权限角色可以触发；不能证明时是否收窄影响范围？
- [ ] 是否证明跨租户影响；不能证明时是否写明没有验证或不适用？
- [ ] 是否证明 source 到 sink 是真实可达路径？
- [ ] 是否存在权限校验、allowlist、escaping、参数数组、固定命令路径把风险阻断？
- [ ] 是否把不可达代码误判成漏洞？
- [ ] 是否把开发环境脚本误判成生产可达漏洞？
- [ ] 是否把依赖 CVE 误判成当前项目可利用漏洞？
- [ ] 是否遗漏异步任务、队列、导入导出、文件转换、备份恢复、插件、webhook、CLI？
- [ ] 是否遗漏参数注入、工具 option 注入、工作目录污染、可执行文件路径污染、配置路径污染？
- [ ] 是否遗漏数据库中用户可写字段被后台任务二次消费？
- [ ] 是否存在任何结论没有复现步骤？

## 4. 判决规则

| 条件 | 判决 |
|---|---|
| 证据完整，可本地复现 | `confirmed` |
| 静态可疑但证据不足 | `candidate` |
| 路径存在但被安全机制阻断 | `blocked` |
| 无法成立、不可达、marker 只是普通回显 | `false_positive` |
| 环境不足，不能下结论 | `needs_environment` |

硬规则：宁可减少 `confirmed` 数量，也不得把 `candidate` 写成 `confirmed`。
