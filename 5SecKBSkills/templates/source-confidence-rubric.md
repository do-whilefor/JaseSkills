# 来源可信度评分模板

| 来源类型 | 基础分 | promoted 资格 | 说明 |
|---|---:|---|---|
| 官方 CVE / MITRE / NVD / CISA KEV | 90-100 | 可 | 仍需版本、影响、修复字段齐全 |
| 厂商官方安全公告 | 90-100 | 可 | 优先级最高之一 |
| GitHub Security Advisory | 80-95 | 可 | 需要确认仓库、版本、修复 |
| 官方 release / patch / commit | 75-95 | 可 | silent fix 推断需 needs_review |
| 平台公开报告 | 60-85 | 条件可 | 需交叉验证 |
| 高质量研究博客 | 50-80 | 条件可 | 需官方或 patch 交叉验证 |
| 社区 PoC 仓库 | 20-60 | 不可单独 promoted | 只能作为候选线索 |
| 标题 / 截图 / 二次转载 / AI 摘要 | 0-30 | 不可 | 不得作为事实来源 |
| 不明来源 | 0-20 | 不可 | 进入 rejected 或 needs_review |

## 冲突处理

1. 厂商公告优先于社区文章。
2. 修复 commit 优先于推测文章。
3. 当前官方规则优先于历史规则。
4. 冲突未解决时，记录必须为 `conflict`。
