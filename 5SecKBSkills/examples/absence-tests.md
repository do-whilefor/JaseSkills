# 缺席测试

删除某个 Skill 后，观察系统丢失的能力，用于确认每个 Skill 是否必要。

| 删除 Skill | 丢失能力 | 是否可由其他 Skill 替代 | 结论 |
|---|---|---|---|
| 01 | 总控调度、边界判断、任务分类 | 不可完全替代 | 必要 |
| 02 | 来源采集、freshness、历史补充 | 不可完全替代 | 必要 |
| 03 | schema、索引、RAG 路由 | 不可完全替代 | 必要 |
| 04 | 漏洞模板保真 | 不可完全替代 | 必要 |
| 05 | 框架级代码审计知识 | 不可完全替代 | 必要 |
| 06 | SRC 合规边界 | 不可完全替代 | 必要 |
| 07 | 工具 release 和误报画像 | 可部分由 02 采集，但不能解读联动 | 必要 |
| 08 | 动态验证和 evidence manifest | 不可替代 | 必要 |
| 09 | patch diff 和变体学习 | 可部分由 04，但深度不足 | 必要 |
| 10 | 反向审计和回归测试 | 不可替代 | 必要 |
