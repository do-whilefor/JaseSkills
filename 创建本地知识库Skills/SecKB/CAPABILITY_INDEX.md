# SecKB Skills 能力索引（v2 审计补强）

本文件是 Claude 进入 SecKB Skills 前的快速能力索引。它不是原文档直接内容，而是**基于文档延伸**的路由辅助层，用于降低误触发、漏触发和跨 Skill 争抢。

## 唯一总控原则

所有涉及 SecKB 的任务，先由 `01-seckb-master-orchestrator` 判断任务类型，再决定是否调用子 Skill。其他 Skill 不应抢先给最终结论。

## 任务类型到 Skill 路由

| 用户任务信号 | 必选 Skill | 可选协作 Skill | 禁止直接输出 |
|---|---|---|---|
| 构建 / 初始化 SecKB | 01 | 03, 10 | 不得声称已联网采集 |
| 更新近 30 天漏洞 / CVE / KEV / GitHub Advisory | 02 | 03, 10 | 不得把新抓取内容 promoted |
| 导入记录 / 建索引 / 修索引 / RAG 查错 | 03 | 10 | 不得伪造索引 |
| 生成漏洞模板 / 修复模板失真 | 04 | 03, 08, 10 | 不得把单 CVE 泛化成通用模板 |
| 按语言或框架做代码审计知识沉淀 | 05 | 04, 08 | 不得把危险函数当漏洞结论 |
| SRC / 众测 / 应急规则 / 可提交性判断 | 06 | 08, 10 | 不得把禁止边界写成执行步骤 |
| 安全工具 release / README / 规则变化 | 07 | 03, 10 | 不得把工具 release 当漏洞证据 |
| 本机 / 靶场 / 授权环境动态验证 | 08 | 04, 06, 10 | 不得越界验证或读取真实敏感数据 |
| patch diff / silent fix / 变体学习 | 09 | 04, 05, 10 | 不得输出武器化链路 |
| 反向审计 / 降级归档 / 负样本测试 | 10 | 01, 03 | 不得美化或伪造审计结果 |

## 触发仲裁规则

1. 如果任务同时像“采集”和“验证”，先调用 01，再调用 02 或 08。验证前必须读取 04/06。
2. 如果任务同时像“工具使用”和“漏洞确认”，先调用 07 记录工具能力，再调用 08 做证据闭环。工具输出不能直接确认漏洞。
3. 如果任务同时像“SRC 规则”和“报告输出”，先调用 06。命中禁止边界时停止动态验证。
4. 如果任务含 CVE、产品、版本，先调用 03 做类型、产品、版本过滤，再调用 04/08。
5. 如果任务模糊，例如“看看有没有洞”，先调用 01 输出需要材料和最小安全路径，不直接调用深度验证。

## 输出熵控制

默认输出只保留五块：结论、依据、映射、缺口、下一步。只有用户要求完整报告或专家模式时，才展开长表格和完整矩阵。

## 反幻觉硬规则

- 不存在的路径、工具、脚本、索引、模板，不得声称存在。
- 无法读取的文件，不得声称已读取。
- 无网络环境下，不得声称已联网更新。
- 未运行脚本，不得声称脚本通过。
- 未做三次稳定复现，不得输出 confirmed。
